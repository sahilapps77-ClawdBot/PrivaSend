The Core Shift

Old Way (Slow & Costly)

We relied on LLMs to "guess" what was sensitive. It was slow, expensive, and non-deterministic.

  • Wait time: 5-10 seconds
  • High API costs
  • "Magic box" - user unsure what happened

New Way (Fast & Precise)

We use robust detection tools (Presidio/Regex) to find everything instantly, then let the user decide.

  • Wait time: < 0.5 seconds
  • Zero cost for detection
  • User sees exactly what is caught

Feature: Smart Bucketing

Instead of hiding the logic, we expose it. We group detections into three buckets so the user knows what to trust.

This builds trust. The user feels in control, not managed by an algorithm.

Feature: "Send to AI"

PrivaSend effectively becomes a secure proxy for your favorite AI tools.

1. User Selects Model

Choose from GPT-4o, Claude 3.5 Sonnet, or Gemini Pro directly in our UI.

2. We Redact First

We strip selected PII from the prompt before it touches the AI provider.

3. Safe Response

You get the AI's answer, but the AI never saw the sensitive data.

Implementation Roadmap

Our plan to execute this pivot in 5 phases.

Phase 1: Backend Config

Disable LLM, Enable Bucketing

Switching off the slow LLM validator. Configuring the new confidence thresholds (0.85 / 0.50).

Phase 2: Integration

Connect OpenRouter

Building the secure bridge to OpenAI, Anthropic, and Google models via a single API.

Phase 3: API Layer

New Endpoints

Creating fast endpoints for /analyze, /redact, and /send-to-ai.

Phase 4: Frontend

The Review UI

Building the interactive "Glass" panel where users toggle categories (Names, IDs, etc.) and confirming redactions.

Phase 5: Deploy

Go Live

Deploying the new lightweight container to our secure VPS.
